## Overview

zendeskopenticketreport is a toolset for generating reports and insights from Zendesk open ticket data. It helps teams monitor, analyze, and act on open tickets efficiently.

- Generate daily and bi-daily open ticket reports
- Suggest improved ticket titles
- Customizable reporting scripts

## Getting Started

### Prerequisites

- Python 3.8 or higher
- pip

### Installation

1. Clone the repository:
	```
	git clone https://github.com/your-org/zendeskopenticketreport.git
	cd zendeskopenticketreport
	```
2. Install dependencies:
	```
	pip install -r requirements.txt
	```

## Usage

### Scripts

- `zendeskreport_opentickets_esc-rarc_bidaily.py`: Generates a bi-daily open tickets report.
- `zendeskreport_opentickets_ticket-title-rename.py`: Suggests improved ticket titles.

### Example

```bash
python zendeskreport_opentickets_esc-rarc_bidaily.py --input tickets.csv --output report.csv
```

## GitHub Actions Workflows

This repository uses GitHub Actions to automate report generation and ticket title recommendations. The workflows are located in `.github/workflows/` and include:

- **zendeskreport_esc-rarc.yml**: 
	- **Purpose:** Automatically generates ESC/RARC open ticket reports.
	- **Triggers:**
		- Scheduled runs on weekdays at 8:00 AM and 4:00 PM Pacific Time (adjusted for daylight saving via UTC cron).
		- Manual runs via the Actions tab.
	- **Steps:**
		1. Checks out the repository.
		2. Sets up Python 3.11.
		3. Installs dependencies from `requirements.txt`.
		4. Runs the report script with required secrets (Zendesk credentials, etc.).

- **zendeskreport_ticket-title-recommendations.yml**:
	- **Purpose:** Generates ticket title suggestions using the relevant script.
	- **Triggers:**
		- Manual runs via the Actions tab.
	- **Steps:**
		1. Checks out the repository.
		2. Sets up Python 3.11.
		3. Installs dependencies from `requirements.txt`.
		4. Runs the title suggestion script with required secrets (Zendesk credentials, Anthropic API key, Google Drive info, etc.).
		5. Uploads the generated report as an artifact for download.

Both workflows use repository secrets to securely provide credentials and API keys. Artifacts generated by the workflows are retained for 30 days.

## Required GitHub Actions Secrets

To run the workflows and scripts, the following GitHub Actions secrets must be configured in your repository:

- `ZENDESK_EMAIL`: Your Zendesk login email (required)
- `ZENDESK_TOKEN`: Zendesk API token (required)
- `ANTHROPIC_API_KEY`: Anthropic API key (required for ticket title recommendations)
- `GDRIVE_SERVICE_ACCOUNT_JSON`: Google service account or OAuth JSON (optional, for Google Drive uploads)
- `GDRIVE_FOLDER_ID`: Target Google Drive folder ID (optional, for Google Drive uploads)
- `GDRIVE_FOLDER_ID_TITLES`: Google Drive folder ID for title suggestions (optional, for title recommendation workflow)

Set these secrets in your repository’s Settings > Secrets and variables > Actions. The workflows and scripts will fail if required secrets are missing.
